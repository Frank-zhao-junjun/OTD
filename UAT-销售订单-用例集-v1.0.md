## UAT-SO-007：风险排序（页内，方案 A）

> **关联 Issue**: DEV-X-02  
> **产品决策**: **方案 A — 接受页内排序**，UI 与审计已明确说明语义  
> **E2E 用例文件**: `e2e/uat-so-007-risk-sort.spec.ts`

### 背景与语义说明

风险排序（优先级排序）为**当前页内重排**：

- SAP 后端按 `SalesOrderDate desc` 返回分页数据（`$skip/$top`）。
- 前端在收到当页数据后，按风险评分（`priorityScore`）对该页行做客户端排序。
- **翻页时**，下一页数据由 SAP 重新返回，前端再对新页做同样的页内排序。
- 风险排序**不跨页全局排序**；全局排序属于方案 B，当前不实现。

此语义已在以下位置明确：

| 位置 | 说明 |
|------|------|
| `src/lib/sap-sales-order-list.ts` | `RISK_SORT_MODE = 'page'`，`RISK_SORT_NOTE` 常量 |
| `src/app/sales-orders/page.tsx` | 查询结果区显示 `RISK_SORT_NOTE` 提示文字 |
| 审计日志 (`logQueryAudit`) | `riskSemantics.mode = 'page'` + `riskSemantics.note` |

### 验收标准

| # | 步骤 | 预期结果 |
|---|------|----------|
| 1 | 打开销售订单页面 `/sales-orders` | 页面正常加载 |
| 2 | 输入任意订单前缀（如 `1`），点击查询 | 返回结果或「暂无数据」 |
| 3 | 检查排序下拉框默认值 | 显示「风险优先级（默认，当页）」 |
| 4 | 有结果时，检查查询结果区标题下方提示文字 | 显示「风险排序仅在当前页内生效；翻页仍按 SAP 基础排序。」 |
| 5 | 检查排序方向下拉框（降序/升序） | **灰显/禁用**，风险排序不允许手动切换方向 |
| 6 | 切换排序为「订单日期」 | 排序方向下拉框变为可用；提示文字**消失** |
| 7 | 再切回「风险优先级」 | 提示文字重新显示；排序方向再次禁用 |

### 不验收项（超出方案 A 范围）

- ❌ 不验证跨页全局风险排名顺序（属方案 B）
- ❌ 不要求第 2 页结果的风险排序与第 1 页形成全局有序序列

### 自动化覆盖

```
e2e/uat-so-007-risk-sort.spec.ts
├── list shows risk column and default risk sort option
├── shows page-level sort note when risk sort is active       ← Plan A 新增
├── sort direction is disabled when risk sort is selected     ← Plan A 新增
├── can switch sort to order date
└── page-level note disappears after switching to non-risk sort ← Plan A 新增
```

## UAT-SO-010：分页翻页

> **关联 Issue**: DEV-X-02（风险排序语义落地，需明确翻页行为）  
> **自动化状态**: 🔴 人工（需要 SAP 沙箱中 >50 条订单数据）

### 背景与语义说明

销售订单列表每页最多显示 50 条（`SALES_ORDER_PAGE_SIZE = 50`）。翻页使用 OData `$skip/$top` 分页，由 SAP 后端控制数据顺序。

**与风险排序的关系（方案 A）**：
- 每次翻页都会重新从 SAP 获取该页数据。
- 若当前排序为「风险优先级」，新页数据到达后**仅在本页范围内**重排。
- 第 2 页的"最高风险"订单不一定比第 1 页的"最低风险"订单风险更高。

### 验收标准

| # | 步骤 | 预期结果 |
|---|------|----------|
| 1 | 在 SAP 沙箱中确保有 >50 条符合默认筛选的订单 | 前置条件：测试数据充足 |
| 2 | 点击查询，等待第 1 页结果加载 | 显示最多 50 条记录；显示总条数或「本页 50 条」 |
| 3 | 点击「下一页」 | 加载第 2 页数据；页码显示变为 2 |
| 4 | 检查第 2 页结果与第 1 页无重复行 | 订单号不重复 |
| 5 | 风险排序模式下，检查第 2 页结果已按风险排序 | 第 2 页内部按风险高→低排列 |
| 6 | 点击「上一页」 | 返回第 1 页，显示与之前相同的结果 |
| 7 | 到达最后一页后，「下一页」按钮禁用 | 确认末页边界处理正确 |

### 注意事项

- 若 SAP 未返回 `@odata.count`，总条数显示「本页 N 条」，不显示「共 X 条」。
- 翻页时若 sortField 为「风险优先级」，排序方向仍显示为禁用（方案 A）。
