## 1. US-MP-01 登录认证 — 测试用例

### 1.1 启动登录态检查

| ID | 场景 | 前置条件 | 操作 | 预期结果 |
|----|------|----------|------|----------|
| T01-01 | 首次启动无 token | Storage 无 `auth_token` | 打开小程序 | 自动跳转登录页 `pages/login/login` |
| T01-02 | 已有有效 token | Storage 有 `auth_token='valid'` | 打开小程序 | 留在当前页，不跳转登录 |
| T01-03 | token 已过期 | Storage 有 `auth_token`，API 返回 401 | 任意 API 请求 | toast "登录已过期，请重新登录" → 2s 后跳转登录页 |
| T01-04 | 多个 API 同时 401 | 同时发起 3 个请求均 401 | 页面加载 | 仅触发一次跳转（防重入锁验证） |

### 1.2 登录页字段校验

| ID | 场景 | 操作 | 预期结果 |
|----|------|------|----------|
| T02-01 | 用户名为空 | 留空用户名，点击登录 | 红色提示"请输入用户名" |
| T02-02 | 用户名过短 | 输入 "ab"，点击登录 | 红色提示"用户名需 3-20 个字符" |
| T02-03 | 用户名含特殊字符 | 输入 "user@123" | 红色提示"用户名只能包含字母、数字、下划线" |
| T02-04 | 密码为空 | 留空密码，点击登录 | 红色提示"请输入密码" |
| T02-05 | 密码过短 | 输入 "12345"，点击登录 | 红色提示"密码需 6-32 个字符" |
| T02-06 | 验证码为空 | 留空验证码，点击登录 | 红色提示"请输入验证码" |
| T02-07 | 验证码非数字 | 输入 "abcd" | 红色提示"验证码为 4 位数字" |
| T02-08 | 全部合法 | 输入合法用户名+密码+4位验证码 | 无红色提示，登录按钮可点击 |

### 1.3 登录交互

| ID | 场景 | 操作 | 预期结果 |
|----|------|------|----------|
| T03-01 | 密码显隐切换 | 点击眼睛图标 | 密码在明文/密文间切换 |
| T03-02 | 验证码刷新 | 点击验证码区域 | 发起 `/api/auth/captcha` 请求，验证码更新 |
| T03-03 | 登录中 loading | 输入合法凭证，点击登录 | 按钮变为 disabled + "登录中..." |
| T03-04 | 登录成功 | 后端返回 `{success, data: {token, user}}` | toast "登录成功" → 1.5s 后跳转首页 |
| T03-05 | 登录失败（凭证错误） | 后端返回 401 或 error | toast "用户名或密码错误" → 验证码自动刷新 + 输入框清空 |
| T03-06 | 网络超时 | 后端无响应 | toast "网络请求失败" |
| T03-07 | 跳转注册 | 点击"去注册 →" | `wx.navigateTo` → `pages/register/register` |
