## 1. 产品愿景

### 1.1 一句话定位
让企业内部员工通过微信小程序**随时随地**查询 SAP ERP 核心业务数据，无需打开电脑或 VPN。

### 1.2 要解决的问题
| 现状痛点 | 解决方案 |
|----------|----------|
| 销售/生产人员在客户现场/车间无法查数据 | 手机即查，4G/5G 可用 |
| Web 端需要浏览器 + VPN | 微信原生，零门槛 |
| 小程序当前只有列表，查不到详情 | 补齐详情页 + 搜索 + Dashboard |
| 不同岗位关注不同数据 | 统一的 Dashboard + 角色自适应预留 |

### 1.3 不解决的问题（Out of Scope）
| 排除项 | 原因 |
|--------|------|
| Excel 导出 | 移动端操作体验差，优先保证查询体验 |
| 数据写入/审批流 | 本产品定位为只读查询，非操作型应用 |
| 离线缓存 | 数据实时性要求高，离线 = 过期数据 |
| 推送通知 | 需要独立的消息队列基础设施 |
| 多语言 | 内部员工工具，简体中文即可 |
| PC 端已有功能的照搬 | 仅补齐移动端高价值场景，不全量迁移 |

## 2. 范围与优先级

### 2.1 优先级逻辑
```
P0 = 移动端核心价值（无此功能则产品不成立）
P1 = 差异化体验（有则好，可延后但不删）
P2 = 链路闭环（从 P0 页面自然延伸）
```

### 2.2 版本策略

| 版本 | 内容 | 目标 |
|------|------|------|
| v1.0 | P0: 登录 + Dashboard + 销售订单详情 + 生产订单详情 | MVP，可上线 |
| v1.1 | P1: 全局搜索 + 视图切换 | 体验优化 |
| v1.2 | P2: 交货单详情 + 开票单据详情 | 链路闭环 |

### 2.3 各版本功能清单

**v1.0 — MVP（当前第1~2批）**

| US | 功能 | 用户价值 |
|----|------|----------|
| US-MP-01 | 登录认证 | 安全基础，防止未授权访问 |
| US-MP-02 | 首页 Dashboard | 一眼看清今日全局 |
| US-MP-03 | 销售订单详情 | 最高频场景：客户现场查订单状态 |
| US-MP-04 | 生产订单详情 | 第二高频：车间查工序和齐套 |

**v1.1 — 体验优化（第3批）**

| US | 功能 | 用户价值 |
|----|------|----------|
| US-MP-05 | 全局搜索 | 记不住单号类型时快速定位 |
| US-MP-06 | 视图切换 | 平板场景查看更密集的数据 |

**v1.2 — 链路闭环（第4批）**

| US | 功能 | 用户价值 |
|----|------|----------|
| US-MP-07 | 交货单详情 | 从销售订单详情自然下钻 |
| US-MP-08 | 开票单据详情 | 财务对账：从销售订单详情跳转 |

## 3. 成功标准

| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 登录成功率 | ≥ 95% | 登录成功数 / 登录尝试数 |
| 首页加载时间 | ≤ 2s | `onLoad` → KPI 渲染完成 |
| 详情页加载时间 | ≤ 3s | 页面进入 → 完整数据渲染 |
| 崩溃率 | < 0.5% | 微信后台性能监控 |
| API 可用性 | 复用 Web 端（不额外要求） | — |

---

## 4. 风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 小程序审核不通过（SAP 数据敏感） | 低 | 高 | 非对外发布，企业内部体验版可绕过 |
| 后端 API 不稳定 | 中 | 中 | 静默降级 + 错误提示 + 重试机制 |
| 微信 API 兼容性 | 低 | 低 | 基础库 ≥ 2.0，不依赖新特性 |
| 业务需求变更 | 中 | 中 | PRD 版本化管理，变更追溯 |
| 人力不足 | 高 | 中 | 按优先级分批交付，P2 可裁剪 |
