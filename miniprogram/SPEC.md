# OTD助手 微信小程序 - 详细规格说明书 (SPEC)

## SPEC-001 登录认证

### 1. 功能概述
用户在小程序启动时进行身份认证，所有业务接口需携带登录态 token。

### 2. 用户场景
> 场景 1：用户首次打开小程序，输入用户名、密码、验证码完成登录。
> 场景 2：用户 token 过期后自动跳转登录页。
> 场景 3：新用户通过注册页创建账号。

### 3. 界面规格

#### 3.1 登录页 (`pages/login/login`)
```
┌─────────────────────────────┐
│  ← 返回                      │
│                             │
│      [SAP Logo]             │
│      OTD助手                │
│      SAP ERP 数据查询系统   │
│                             │
│  ┌─────────────────────┐   │
│  │ 👤 用户名            │   │
│  └─────────────────────┘   │
│  ┌─────────────────────┐   │
│  │ 🔒 密码              │   │
│  └─────────────────────┘   │
│  ┌──────────┐ ┌───────┐   │
│  │ 验证码输入 │ │[验证码]│   │
│  └──────────┘ └───────┘   │
│                             │
│  ┌─────────────────────┐   │
│  │      登 录           │   │
│  └─────────────────────┘   │
│                             │
│      还没有账号？去注册 →   │
│                             │
└─────────────────────────────┘
```

**字段校验**：
| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 用户名 | string | 是 | 3-20 字符，字母数字下划线 |
| 密码 | string | 是 | 6-32 字符 |
| 验证码 | string | 是 | 4 位数字 |

**交互**：
- 密码输入框右侧显示眼睛图标，点击切换明文/密文
- 验证码图片点击刷新，请求 `/api/auth/captcha`
- 登录按钮 loading 状态，提交中禁用
- 校验失败显示错误提示（字段下方红色文字）

#### 3.2 注册页 (`pages/register/register`)
同登录页布局，增加"确认密码"字段。

**确认密码校验**：必须与密码一致。

### 4. 接口调用

#### 4.1 获取验证码
```
GET /api/auth/captcha
Response: { success: true, data: { svg: "<svg...>", text: "1234" } }
```
**注意**：小程序不支持直接渲染 SVG，需要后端返回 base64 PNG 或将 SVG 转为图片 URL。或改用纯数字验证码文字展示。

#### 4.2 登录
```
POST /api/auth/login
Body: { username: string, password: string, captcha: string }
Response: { success: true, data: { token: string, user: { id, username } } }
```

#### 4.3 注册
```
POST /api/auth/register
Body: { username: string, password: string, confirmPassword: string, captcha: string }
```

#### 4.4 获取当前用户
```
GET /api/auth/me
Header: x-session: <token>
Response: { success: true, data: { id, username } }
```

### 5. 状态管理
- Token 存储：`wx.setStorageSync('auth_token', token)`
- Token 读取：每次 API 请求从 storage 取出放入 `x-session` header
- Token 过期：后端返回 401，前端清除 token 跳转登录页
- 启动检查：`app.js onLaunch` 中检查 token，无则跳转登录页

### 6. 错误处理
| 错误场景 | 处理方式 |
|----------|----------|
| 用户名不存在 | 提示"用户名或密码错误"（模糊提示） |
| 密码错误 | 同上 |
| 验证码错误 | 提示"验证码错误"，自动刷新验证码 |
| 网络超时 | 提示"网络连接失败，请重试" |

### 7. 验收标准
- [ ] 未登录时打开小程序自动跳转登录页
- [ ] 输入正确凭证后 1s 内完成登录并跳转首页
- [ ] 密码错误时验证码自动刷新
- [ ] token 过期后再次请求 API 自动跳转登录页
- [ ] 注册成功后自动登录并跳转首页

## SPEC-002 首页 Dashboard

> 场景 2：用户下拉刷新，更新最新数据。

- 尺寸：宽度 48%，高度 100px
- 背景：白色，圆角 8px，阴影
- 颜色标识：销售(蓝 #0A6ED1)、生产(绿 #107E3E)、待发货(橙 #E9730C)、库存(紫)
- 图标：左侧 40px 圆形图标

**活动流样式**：
- 左侧 4px 竖条：成功(绿)、警告(橙)、错误(红)
- 每行高度 60px

### 4. 接口调用
```
GET /api/dashboard
Header: x-session: <token>
Response: {
  success: true,
  data: {
    kpis: [
      { label: "销售订单", value: "19", unit: "张", color: "blue", icon: "FileText" },
```

- 页面 `onPullDownRefresh`：下拉刷新重新获取 dashboard 数据
- KPI 卡片点击跳转对应列表页
- 模块图标点击跳转对应列表页

- [ ] 首页加载 2s 内显示所有 KPI 数据

**步骤 1：获取订单详情**
```
GET /api/sap/CE_SALESORDER_0001/SalesOrder?id=1&expand=_Item,_Partner,_PricingElement
```

**步骤 2：获取关联单据**
```
GET /api/sap/sales-order/1/related
Response: {
  deliveryByItem: { "10": [...] },
  billingByItem: { "10": [...] },
  productionOrders: [...]
}
```

### 5. 字段映射

| 显示字段 | 数据源 | 格式化 |
|----------|--------|--------|
| 订单号 | SalesOrder | - |
| 类型 | SalesOrderType | - |
| 客户 | SoldToParty | - |
| 订单日期 | SalesOrderDate | YYYY-MM-DD |
| 总金额 | TotalNetAmount | 千分位 + 货币 |
| 状态 | OverallSDProcessStatus | C→已完成 A→处理中 B→部分完成 |

**行项目映射**：
| 字段 | 数据源 |
|------|--------|
| 物料 | Material |
| 描述 | SalesOrderItemText |
| 数量 | OrderQuantity |
| 单位 | OrderQuantityUnit |
| 单价 | NetPrice |
| 净额 | NetAmount |
| 已确认数量 | ConfdDelivQtyInOrderQtyUnit |

### 6. 交互
- 点击交货单号 → 跳转交货单详情页
- 点击开票单号 → 跳转开票单据详情页
- 点击生产订单号 → 跳转生产订单详情页
- 定价明细可折叠（默认折叠）

### 7. 验收标准
- [ ] 页面加载 3s 内显示完整数据
- [ ] 所有关联单据正确显示
- [ ] 状态颜色正确（已完成=绿、处理中=蓝、部分=橙）

## SPEC-005 全局搜索

跨 8 大 SAP 实体实时搜索，按类型分组展示结果。

### 2. 用户场景
> 场景：用户只记得一个订单号片段"1000"，不确定是销售订单还是生产订单，通过全局搜索快速定位。

### 3. 界面规格

```
┌─────────────────────────────┐
│  🔍 搜索...         [取消]   │  ← 搜索输入框
├─────────────────────────────┤
│  搜索历史                    │
│  [1000] [FG10] [200000] [x] │  ← 标签式历史记录
├─────────────────────────────┤
│  📦 销售订单 (3)            │
│  订单 1 | 客户: 200000      │
│  订单 10 | 客户: 200000     │
│  订单 19 | 客户: 200000     │
│                             │
│  🏭 生产订单 (2)            │
│  1000142 | FG52             │
│  1000000 | FG10             │
│                             │
│  🚚 交货单 (3)              │
│  80000005 | 客户: 1000001   │
│  ...                        │
└─────────────────────────────┘
```

### 4. 接口调用
```
GET /api/search?q=1000
Header: x-session: <token>
Response: {
  success: true,
  query: "1000",
  results: [
    {
      group: "销售订单",
      icon: "FileText",
      items: [
        { label: "订单 1", description: "客户: 200000", path: "/sales-orders/1" }
      ]
    }
  ]
}
```

### 5. 交互
- 输入框聚焦自动弹出搜索历史
- 输入字符 300ms 防抖后触发搜索
- 点击搜索结果跳转对应详情页
- 搜索历史存储到本地，最多 10 条，可单独删除

### 6. 验收标准
- [ ] 输入后 500ms 内显示搜索结果
- [ ] 结果按实体类型正确分组
- [ ] 搜索历史正确记录和展示
- [ ] 空结果提示"未找到匹配数据"
