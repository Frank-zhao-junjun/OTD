# DESIGN.md

## 设计思考

### 产品定位
SAP ERP数据查询助手，面向企业内部用户，提供产品、销售订单、生产订单、库存等核心业务数据的快速查询能力。

### 气质与意象
- **关键词**：专业、清晰、高效、数据驱动
- **具象场景**：企业办公环境，用户在简洁的工作台前快速检索业务数据，屏幕上是整洁的数据表格，没有多余装饰，信息一目了然

### 配色方案
- **主色**：中性灰蓝调（企业级系统常用色，专业稳重）
- **功能色**：
  - 蓝色：主要操作按钮、链接
  - 绿色：成功状态、正向数据
  - 红色：警告/错误状态
  - 橙色：待处理状态

### 字体排版
- **字体族**：系统默认字体栈（确保中英文兼容）
- **表格数据**：紧凑排列，适度行高
- **标题层级**：清晰区分，用字号和字重而非颜色

### 页面结构
- **导航栏**：顶部固定，包含模块切换（产品、订单、生产、库存）
- **查询区**：搜索框 + 筛选条件 + 查询按钮
- **数据区**：表格展示查询结果，支持分页
- **详情区**：点击行展开详情或侧边面板显示

### 组件规范
- 表格：紧凑型，带边框，支持排序和筛选
- 按钮：方形或微圆角，无过多装饰
- 输入框：简洁边框，聚焦时有明显反馈
- 卡片：用于详情展示，轻阴影

### 交互与状态
- 加载状态：表格区域显示骨架屏或加载动画
- 空状态：友好提示，引导用户操作
- 错误处理：明确提示错误原因和解决建议

### 设计禁忌
- 禁止花哨渐变和过度装饰
- 禁止复杂动画影响数据查看效率
- 禁止颜色过多导致视觉混乱
- 禁止信息层级不清晰
